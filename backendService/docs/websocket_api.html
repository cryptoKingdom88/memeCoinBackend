WebSocket API Documentation

Overview

The Backend WebSocket Service provides real-time cryptocurrency data streaming through WebSocket connections. Clients can subscribe to the "dashboard" channel to receive live updates about token information, trade data, and aggregated metrics.

Connection

Endpoint: ws://localhost:8080/ws
Protocol: WebSocket (RFC 6455)
Format: JSON messages with UTF-8 encoding

Quick Start

1. Connect to WebSocket

const ws = new WebSocket('ws://localhost:8080/ws');

2. Subscribe to Dashboard

ws.send(JSON.stringify({
    "type": "subscribe",
    "channel": "dashboard"
}));

3. Receive Real-time Data

ws.onmessage = function(event) {
    const message = JSON.parse(event.data);
    console.log('Received:', message.type, message.data);
};

Message Types

Token Information (token_info)

Broadcasted when new token information is available.

{
  "type": "token_info",
  "channel": "dashboard",
  "data": {
    "token_address": "0x1234567890abcdef",
    "symbol": "ETH",
    "name": "Ethereum",
    "price": 2500.50,
    "market_cap": 300000000000
  },
  "timestamp": "2024-01-01T12:00:00Z"
}

Trade Data (trade_data)

Broadcasted for each new trade transaction.

{
  "type": "trade_data",
  "channel": "dashboard",
  "data": {
    "token_address": "0x1234567890abcdef",
    "price": 2500.50,
    "volume": 1000.0,
    "timestamp": "2024-01-01T12:00:00Z",
    "tx_hash": "0xabcdef1234567890"
  },
  "timestamp": "2024-01-01T12:00:00Z"
}

Aggregate Data (aggregate_data)

Broadcasted with aggregated trading metrics over time windows.

{
  "type": "aggregate_data",
  "channel": "dashboard",
  "data": {
    "token_address": "0x1234567890abcdef",
    "total_volume": 50000.0,
    "average_price": 2500.50,
    "trade_count": 100,
    "time_window": "1h",
    "window_start": "2024-01-01T12:00:00Z",
    "window_end": "2024-01-01T13:00:00Z"
  },
  "timestamp": "2024-01-01T12:00:00Z"
}

Client Examples

JavaScript/Browser

const ws = new WebSocket('ws://localhost:8080/ws');

ws.onopen = function() {
    console.log('Connected to WebSocket');
    ws.send(JSON.stringify({
        type: 'subscribe',
        channel: 'dashboard'
    }));
};

ws.onmessage = function(event) {
    const message = JSON.parse(event.data);
    switch(message.type) {
        case 'token_info':
            handleTokenInfo(message.data);
            break;
        case 'trade_data':
            handleTradeData(message.data);
            break;
        case 'aggregate_data':
            handleAggregateData(message.data);
            break;
    }
};

Python

import asyncio
import websockets
import json

async def client():
    uri = "ws://localhost:8080/ws"
    async with websockets.connect(uri) as websocket:
        await websocket.send(json.dumps({
            "type": "subscribe",
            "channel": "dashboard"
        }))
        
        async for message in websocket:
            data = json.loads(message)
            print(f"Received: {data['type']}", data['data'])

asyncio.run(client())

Error Handling

Error Type Description Response
Invalid Channel Subscription to unsupported channel {"error": "Invalid subscription channel", "code": 400}
Connection Lost Client disconnected unexpectedly Automatic cleanup, no response
Invalid JSON Malformed message sent to server Error message sent to client

Performance & Limits

  • Maximum Clients: 1000 concurrent connections
  • Message Buffer: 100 messages per client
  • Supported Channels: "dashboard" only
  • Message Rate: No explicit rate limiting
